Плагин Affix

Позволяет зафиксировать элемент в конкретном месте на странице в зависимости от величины прокрутки. Этот эффект очень часто используется для фиксирования меню или значков социальных сетей во время прокрутки остального содержимого. Дынный режим включается/выключается путем изменения CSS свойства position на static или fixed

С помощью аттрибутов

Если к элементу добавить атрибут data-spy="affix" , то он автоматически будет менять свое CSS свойство position на fixed в начале прокрутки Документа вверх. Для того чтобы задать отступ в пикселях от начала Документа и тем самым задержать приведение position к fixed, используется дополнительный атрибут data-offset-top, значением которого буде число пикселей без добавления px.

До прокрутки плагин добавляет к выбраному элементу класс .affix-top. Изменение CSS свойство position на fixed происходит путем замены вышеуказанного класса на .affix, который уже определен в файле bootstrap.css

При прокрутке Документа вниз в той же точке происходит обратная замена вышеуказанных классов

Если внизу Документа находится элемент или блок, который никак не соотносится с зафиксированным элементом и может его перекрыть, тогда фиксацию необходимо снять и позволить элементу двигаться с остальным содержимым. Это достигается путем замены плагином класса .affix на класс .affix-bottom, в котором CSS свойство position лучше установить в значение absolute. При этом необходимо задать отступ в пикселях (без px) от нижнего края Документа с помощью дополнительного атрибута data-offset-bottom, который обозначит точку переключения классов.

С помощью JavaScript

Для того чтобы программным образом придать какому-то элементу свойства affix, этот элемент необходимо перевести в объект JQuery , а затем вызвать функцию affix() с параметром в виде объекта. имеющего свойство offset

$('#fix').affix({
offset: {
top: Number or Function,
bottom: Number or Function
}
});

The .offset() method allows us to retrieve the current position of an element relative to the document. Contrast this with .position(), which retrieves the current position relative to the offset parent. When positioning a new element on top of an existing one for global manipulation (in particular, for implementing drag-and-drop), .offset() is more useful. .offset() returns an object containing the properties top and left. Note: jQuery does not support getting the offset coordinates of hidden elements or accounting for borders, margins, or padding set on the body element. While it is possible to get the coordinates of elements with visibility:hidden set, display:none is excluded from the rendering tree and thus has a position that is undefined. Additional Notes: The number returned by dimensions-related APIs, including .offset(), may be fractional in some cases. Code should not assume it is an integer. Also, dimensions may be incorrect when the page is zoomed by the user; browsers do not expose an API to detect this condition.
